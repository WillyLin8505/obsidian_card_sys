# 系統診斷指南

## 🎯 目的

系統診斷工具幫助您快速檢測和定位應用程式中的問題，特別是在 UUID 遷移後。

## 📍 如何訪問

在應用程式的側邊欄中，點擊「**系統診斷**」連結，或直接訪問：
```
/diagnostic-test
```

## 🧪 測試項目

診斷工具會自動運行以下測試：

### 1. **資料庫連接測試**
- ✅ 檢查 Supabase 資料庫是否可訪問
- ✅ 驗證 API 端點是否正常工作

### 2. **獲取筆記測試**
- ✅ 測試從資料庫獲取所有筆記
- ✅ 顯示當前筆記總數

### 3. **創建筆記測試**
- ✅ 自動創建一個測試筆記
- ✅ 驗證 UUID 生成是否正確

### 4. **獲取單個筆記測試**
- ✅ 通過 ID 獲取筆記
- ✅ 驗證數據完整性

### 5. **更新筆記測試**
- ✅ 修改筆記標題和內容
- ✅ 驗證更新功能正常

### 6. **創建連結測試**
- ✅ 創建筆記之間的連結
- ✅ 測試雙向連結功能

### 7. **搜尋筆記測試**
- ✅ 測試全文搜尋功能
- ✅ 驗證搜尋結果準確性

### 8. **刪除筆記測試**
- ✅ 軟刪除測試筆記
- ✅ 確保測試後清理

### 9. **UUID 格式驗證**
- ✅ 檢查所有筆記 ID 是否為有效的 UUID v4 格式
- ✅ 識別任何舊格式的 ID（如果存在）

## 🚀 使用步驟

### 步驟 1: 運行測試
1. 導航到「系統診斷」頁面
2. 點擊「**開始測試**」按鈕
3. 等待所有測試完成（通常需要 5-10 秒）

### 步驟 2: 查看結果
測試完成後，您會看到：
- **總測試數**：運行的測試總數
- **通過**：成功通過的測試數量（綠色）
- **失敗**：失敗的測試數量（紅色）

### 步驟 3: 診斷問題
對於任何失敗的測試：
1. 點擊測試卡片
2. 展開「**查看詳細信息**」
3. 查看錯誤消息和詳細數據
4. 根據錯誤類型採取相應行動

## ⚠️ 常見問題和解決方案

### 問題 1: 資料庫連接失敗
**錯誤消息**: `Failed to connect to database`

**解決方案**:
1. 檢查您的網絡連接
2. 確認 Supabase 項目正在運行
3. 驗證環境變量是否正確設置

### 問題 2: 創建筆記失敗
**錯誤消息**: `invalid input syntax for type uuid`

**解決方案**:
1. 資料庫表可能仍然使用舊的 schema
2. 前往「資料庫管理」頁面
3. 執行資料庫重置或重新運行遷移腳本

### 問題 3: UUID 驗證失敗
**錯誤消息**: `發現 X 個無效的 UUID`

**解決方案**:
1. 您的資料庫中仍有舊格式的筆記 ID
2. 需要運行資料遷移工具轉換舊數據
3. 或者清空資料庫重新開始

### 問題 4: 獲取筆記失敗
**錯誤消息**: `column "content" does not exist`

**解決方案**:
1. 資料庫 schema 不正確
2. 需要重新執行 SQL 遷移腳本
3. 確保使用最新版本的 schema

### 問題 5: 搜尋功能失敗
**錯誤消息**: 搜尋相關錯誤

**解決方案**:
1. 檢查 `note_chunks` 表是否存在
2. 確保筆記有對應的 chunks
3. 驗證搜尋索引是否正確

## 🛠️ 進階調試

如果診斷測試仍然失敗，您可以：

### 1. 檢查瀏覽器控制台
打開開發者工具 (F12) 並查看 Console 標籤，尋找詳細的錯誤日誌。

### 2. 檢查網絡請求
在開發者工具的 Network 標籤中：
- 查看 API 請求是否成功
- 檢查響應狀態碼
- 查看響應數據

### 3. 檢查 Supabase 日誌
1. 前往 Supabase Dashboard
2. 進入 Logs 部分
3. 查看 Edge Functions 日誌
4. 尋找錯誤消息

### 4. 驗證資料庫表結構
在 Supabase SQL Editor 中運行：
```sql
-- 檢查 notes 表結構
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'notes';

-- 檢查 ID 類型
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'notes' AND column_name = 'id';
-- 應該返回: id | uuid
```
